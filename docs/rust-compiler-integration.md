According to a document from **May 7, 2026**, the current Chimera repo already has a Rust tools workspace with `chimera-adapter-rust`, `chimera-build`, `chimera-cli`, `chimera-meta`, `chimera-object`, `chimera-wrappergen`, `chimera-cache`, and `chimera-proof-bridge`; the existing Rust adapter is currently scoped to `repr(C)`, extern blocks, forbidden native Rust types, and FFI safety validation, not full Rust MIR/body lowering to ChimeraIR.  

For Rust compiler integration, do **not** create a separate “rustmera” repo yet. Keep Rust lowering inside the main `chimera/tools/crates/` workspace and add a `chimera-rust-*` crate family there. The Rust compiler itself should **not** be vendored. Use the installed Rust toolchain through Cargo/rustc, and add an optional `rustc_driver`-based sidecar crate for semantic extraction. The official rustc-dev-guide says `rustc_driver` is rustc’s main driver/glue layer and recommends it where possible over using `rustc_interface` directly; external drivers using compiler internals require `rustc_private`, plus `rustc-dev` and `llvm-tools` components on official toolchains. ([rustc-dev-guide.rust-lang.org][1])

The intended flow should be:

```text
Cargo / rustc
  → chimera-rustc-driver or stable surface parser
  → .rsnap semantic snapshot
  → .rdep dependency graph
  → .rmirpack typed MIR/HIR/type/layout package
  → chimera-rust-dialect
  → chimera-rust-to-chimera
  → .chimera / Chimera MLIR
  → compiler-core chimerac
  → .chmeta / .cho / .cproof / .chwrap
  → Lean proof bridge
  → chimera link
```

The existing documented artifact flow already expects frontends for C/Rust/Zig to emit ChimeraIR consumed by `compiler-core`, which then emits objects, metadata, proof artifacts, wrappers, and final linked binaries.

## Completion rule

A row can move to **Complete** only when the implementation is merged in the owning crate or compiler-core, the listed tests pass in CI, and the listed documentation exists and is linked from the Rust design docs. The current repo has a useful Rust FFI validation scaffold, but full Rust→ChimeraIR completion requires the new compiler extraction, MIR import, Rust dialect, ChimeraIR/MLIR lowering, metadata, wrapper, cache, and proof layers above.
